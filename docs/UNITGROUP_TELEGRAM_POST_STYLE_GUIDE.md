# 📰 UNITGROUP TELEGRAM POST STYLE GUIDE
## Полное руководство по формату постов для @UnitgroupAI

**Версия:** 1.0  
**Дата:** 2026-07-14  
**Канал:** @UnitgroupAI  
**Позиционирование:** Полезное деловое медиа для производства  

---

## 1. ПОЗИЦИОНИРОВАНИЕ

Канал публикует:
- ✅ мини-уроки по производству;
- ✅ бизнес-разборы материалов и техники;
- ✅ полезные инструкции;
- ✅ объяснение сложных тем;
- ✅ материалы, которые хочется сохранить.

Канал НЕ публикует:
- ❌ копии RSS-новостей;
- ❌ переводы всех статей;
- ❌ технические логи;
- ❌ рекламу неготовых продуктов;
- ❌ случайный набор текста;
- ❌ сырой вывод feedparser.

---

## 2. СТРУКТУРА ПОСТА

Универсальная формула (не обязательны все блоки):

```
ЗАГОЛОВОК С ЭМОДЗИ
↓
ПРОБЛЕМА (2–5 абзацев)
↓
ОБЪЯСНЕНИЕ (если нужно)
↓
СРАВНЕНИЕ (обычно необходимо)
↓
РАСЧЁТ или ПРИМЕР (если есть данные)
↓
УСЛОВНЫЙ КЕЙС (если применимо)
↓
ВЫГОДНО / НЕВЫГОДНО
↓
ВАРИАНТЫ ДЕЙСТВИЙ
↓
ВЫВОД (2–4 предложения)
↓
ВОПРОС АУДИТОРИИ (1–2 вопроса)
↓
ХЭШТЕГИ (5–10)
```

---

## 3. ЗАГОЛОВОК

**Формат:**
```
🌱 ЭКОПЛАСТИК vs ОБЫЧНЫЙ ПЛАСТИК — ЧТО ВЫБРАТЬ В 2026?
```

**Правила:**
- 1 релевантный эмодзи в начале;
- отдельная строка;
- допустимы заглавные буквы;
- сравнение или вопрос;
- без фейкового кликбейта;
- без «ШОК», «СЕКРЕТ», «100%»;
- без гарантии прибыли;
- без неподтверждённых процентов.

**Примеры формул:**
```
⚙️ НОВЫЙ ИЛИ Б/У СТАНОК — ЧТО ВЫГОДНЕЕ ДЛЯ ЦЕХА?

🪵 ЛДСП vs МДФ — ГДЕ ПРОИЗВОДИТЕЛЬ ТЕРЯЕТ ДЕНЕГ?

🔩 ЛАЗЕРНАЯ РЕЗКА vs ПЛАЗМА — ЧТО ВЫБРАТЬ ДЛЯ МАЛОГО ПРОИЗВОДСТВА?

💡 СВОЁ ИЗДЕЛИЕ ИЛИ РАБОТА ПОД ЗАКАЗ — ГДЕ БОЛЬШЕ ВОЗМОЖНОСТЕЙ?
```

---

## 4. БЛОК ПРОБЛЕМА

**Формат:**
```
❗ Проблема

Вы производите мебель из пластика.

Клиент просит использовать экологичный материал.

Вы не понимаете, насколько это увеличит себестоимость.
```

**Правила:**
- ❗ эмодзи и заголовок;
- 2–5 коротких абзацев;
- обращение через «вы»;
- реальная ситуация;
- читатель узнает свою проблему;
- без длинного вступления.

---

## 5. БЛОК ОБЪЯСНЕНИЕ

**Формат:**
```
🧩 Что такое экопластик?

Обычный материал:
• на основе нефти;
• разлагается 200–500 лет;
• стоит дешевле.

Экологичный материал:
• на основе растений;
• разлагается 2–5 лет;
• стоит дороже.
```

**Правила:**
- 🧩 эмоджи и краткий вопрос;
- короткие пункты;
- 3–5 пунктов каждого варианта;
- одна мысль в строке;
- объяснять сложные термины;
- не смешивать разные понятия;
- только подтверждённые факты.

---

## 6. БЛОК СРАВНЕНИЕ

**Формат:**
```
📊 СРАВНЕНИЕ ПО КЛЮЧЕВЫМ ПАРАМЕТРАМ

💰 ЦЕНА

Обычный материал:
• ~100 ₽/кг;
• доступны все поставщики.

Экологичный:
• ~150–180 ₽/кг;
• узкий круг поставщиков.

Разница:
• на каждый килограмм +50–80 ₽.
```

**Допустимые категории:**
- 💰 Цена
- 🌍 Экология
- 🛠 Прочность
- ♻️ Переработка
- ⏱ Срок службы
- 🏭 Производство
- 📦 Логистика
- 🔥 Термостойкость
- ⚡ Энергия
- 🎯 Производительность

**Правила:**
- заголовок с 📊;
- подзаголовок каждого параметра;
- структурированное сравнение;
- не использовать Markdown-таблицы;
- по 2–3 пункта на вариант.

---

## 7. БЛОК РАСЧЁТ

**Формат:**
```
🧮 УСЛОВНЫЙ ПРИМЕР

Допустим, изделие требует 20 кг материала.

Обычный материал:
20 кг × 100 ₽ = 2 000 ₽

Более дорогой материал:
20 кг × 150 ₽ = 3 000 ₽

Разница:
1 000 ₽ на изделие.

Это условный пример для понимания логики. 
Для реального расчёта нужны актуальные цены поставщика.
```

**Правила:**
- 🧮 эмодзи;
- показывать исходные данные;
- показывать формулу;
- показывать результат;
- НЕ выдумывать реальные цены;
- НЕ давать финансовые гарантии;
- обязательная пометка об условном характере;
- не советовать закупать тонны без анализа.

---

## 8. БЛОК КЕЙС

**Разрешённые форматы:**

```
🏭 ПРАКТИЧЕСКИЙ ПРИМЕР
```

или:

```
📌 УСЛОВНЫЙ КЕЙС
```

**Запрещённые формулировки:**

❌ «Реальная фабрика из Петербурга увеличила продажи на 40%»

без:
- названия фабрики;
- ссылки на публикацию;
- подтверждения.

**Если кейс вымышленный, писать:**

✅ Условный пример мебельного производства.

**Структура:**
```
Было:
• обычный пластик;
• цена закупки 50 000 ₽/месяц.

Изменили:
• перешли на биопластик;
• переподготовили станки.

Возможный результат:
• появились клиенты с зелёными стандартами;
• маржа временно снизилась.
```

**Правила:**
- не выдавать расчёт за доказанный кейс;
- использовать слова: может, зависит, требует, разумно;
- не использовать: точно, гарантированно, обеспечено.

---

## 9. БЛОК ВЫГОДНО / НЕВЫГОДНО

**Формат:**
```
✅ КОГДА ЭТО МОЖЕТ БЫТЬ ВЫГОДНО

• Клиенты платят за экологичность.
• Вы ориентированы на премиум-сегмент.
• Ваш объём позволяет амортизировать закупку.

❌ КОГДА ЭТО МОЖЕТ БЫТЬ НЕВЫГОДНО

• Цена для клиента критична.
• Нет гарантии спроса на эко-товары.
• Поставщик нестабилен.
```

**Правила:**
- ✅ и ❌ эмодзи для обоих блоков;
- 3–5 пунктов каждый;
- мягкие формулировки: может быть, зависит, требует;
- без абсолютов: точно, гарантия, обеспечено.

---

## 10. БЛОК ВАРИАНТЫ ДЕЙСТВИЙ

**Формат:**
```
🛠 ЧТО МОЖНО СДЕЛАТЬ

Вариант 1. Оставить текущий материал
Подходит, если клиенты выбирают по цене.

Вариант 2. Сделать тестовую линейку
Проверить спрос на небольшой партии.

Вариант 3. Предложить выбор клиенту
Standard и Eco с понятной разницей в цене.
```

**Правила:**
- 🛠 эмодзи;
- 2–4 варианта;
- каждый вариант отдельный абзац;
- краткое объяснение;
- без гарантии результата;
- без рекламы неготовых сервисов.

---

## 11. БЛОК ВЫВОД

**Формат:**
```
🎯 ВЫВОД

Экологичный материал не становится выгодным только из-за слова «эко».

Переход имеет смысл, если свойства подходят товару, 
поставщик надёжен, а клиент понимает ценность.
```

**Правила:**
- 🎯 эмодзи;
- 2–4 предложения;
- практическая мысль;
- без повторения всего поста;
- без рекламы продукта;
- логичное завершение.

---

## 12. БЛОК ВОПРОС АУДИТОРИИ

**Формат:**
```
💬 Вы уже тестировали экопластик?

Что оказалось сложнее: цена, качество или поставщики?
```

**Правила:**
- 💬 эмодзи;
- 1–2 вопроса;
- конкретная тема;
- НЕ CTA на Mini App;
- НЕ предложение купить;
- НЕ одинаковый вопрос в каждом посте;
- поощрять обсуждение.

---

## 13. ХЭШТЕГИ

**Формат:**
```
#экопластик #пластик #материалы #производство #экология #мебель #бизнес
```

**Правила:**
- 5–10 хэштегов;
- только релевантные;
- не повторять один набор во всех постах;
- не вставлять между абзацами;
- не добавлять популярные случайные теги;
- максимум две строки;
- в конце поста.

## 15. ПРАВИЛА ОТСТУПОВ

**Обязательное:**

1. ✅ После заголовка — одна пустая строка.
2. ✅ Между разделами — одна пустая строка.
3. ✅ Не более двух пустых строк подряд.
4. ✅ Абзац — 2–3 предложения максимум.
5. ✅ Списки через `•`.
6. ✅ Разделы отдельной строкой.

**Запрещённое:**

1. ❌ Длинное полотно без пауз.
2. ❌ Markdown-таблицы (используем пункты).
3. ❌ JSON или Python dict.
4. ❌ HTML-теги.
5. ❌ RSS metadata и сырые структуры.

---

## 16. ДЛИНА ПОСТА

**Целевая:** 1 500–3 500 символов.

**Допустимо:** до 4 000 для сложного материала.

**Правила:**
- не копировать полный текст статьи;
- не обрезать предложение посередине;
- не сокращать хороший текст ради картинки;
- лучше убрать один раздел, чем сломать структуру.

---

## 17. ФАКТЧЕКИНГ

**Каждое утверждение имеет статус:**

- `verified` — подтверждено, можно публиковать;
- `conditional_example` — только с пометкой «условный пример»;
- `estimate` — с диапазоном и пояснением;
- `unverified` — запрещено.

**Запрещено без подтверждения:**

❌ конкретные цены;
❌ прогнозы цен;
❌ гарантия роста прибыли;
❌ точные сроки разложения;
❌ количество переработок;
❌ результат несуществующей фабрики;
❌ будущие действия OPEC;
❌ неподтверждённые проценты.

## 20. ПРИМЕРЫ ПЛОХИХ ПОСТОВ

❌ Пример 1 — Копия RSS:

> Компания X выпустила новый станок. Станок может резать металл со скоростью 100 м/мин. 
> Цена 5 млн рублей. Доставка в течение 30 дней. Обратитесь к нашему менеджеру.

**Проблема:** Это объявление, не полезный контент.

---

❌ Пример 2 — Сырая структура feedparser:

> {'type': 'text/html', 'language': None, 'base': 'https://example.com', 
> 'value': '<p>Some HTML content</p>'}

**Проблема:** Техническая структура, не пост.

---

❌ Пример 3 — Вымышленный кейс без пометки:

> На одной фабрике использовали экопластик и заработали на 500% больше.

**Проблема:** Без названия и подтверждения это выглядит как ложь.

---

❌ Пример 4 — Реклама неготовых продуктов:

> Откройте нашу Mini App и рассчитайте заказ за 30 секунд! 
> Получите коммерческое предложение прямо сейчас!

**Проблема:** Это продажа, не контент.

## 21. ЧЕК-ЛИСТ ПЕРЕД ПУБЛИКАЦИЕЙ

Перед тем как опубликовать пост, проверить:

```
Структура:
☐ Есть заголовок с эмодзи
☐ Есть проблема (2–5 абзацев)
☐ Есть объяснение или сравнение
☐ Есть вывод (2–4 предложения)
☐ Есть вопрос аудитории
☐ Есть хэштеги

Текст:
☐ На русском языке
☐ 1 500–4 000 символов
☐ Абзацы по 2–3 предложения
☐ Отступы между разделами
☐ Нет HTML-тегов
☐ Нет Python dict/JSON
☐ Нет RSS metadata

Факты:
☐ Все цифры подтверждены
☐ Примеры помечены как «условные»
☐ Нет гарантий прибыли
☐ Нет выдуманных кейсов

Реклама:
☐ Нет UNITGROUP/UNITPLAST упоминаний
☐ Нет Mini App CTA
☐ Нет КП/демо-заявок
☐ Это полезный контент, не продажа

Эмодзи и хэштеги:
☐ 1 эмодзи в заголовке
☐ 1 эмодзи на крупный раздел
☐ 5–10 релевантных хэштегов
☐ Хэштеги в конце

Технически:
☐ Нет сырых feedparser структур
☐ Нет обрезанных предложений
☐ Нет лишних пустых строк
```
