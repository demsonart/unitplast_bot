## 📊 IMPLEMENTATION DETAILS

### Image Source Types
```
1. "rss"              - From RSS feed articles
2. "telegram"         - From Telegram posts
3. "ai_generated"     - Generated by AI (DALL-E, Midjourney)
4. "official_press"   - Official press releases (when allowed)
```

### Approval Statuses
```
- "needs_review"      - Awaiting admin review
- "approved"          - Approved for use
- "rejected"          - Rejected by admin
- "pending_generation"  - Waiting for AI generation
```

### Visual Prompt Structure
```json
{
  "text": "Detailed prompt in Russian",
  "language": "ru",
  "style": "technical|realistic|illustrated|abstract",
  "quality": "low|medium|high",
  "aspect_ratio": "16:9",
  "themes": ["theme1", "theme2", "theme3"]
}
```

### Workflow Integration

```
POST CREATION
     ↓
SOURCE SELECTION (RSS or Telegram)
     ↓
IMAGE PARSING
├─ RSS: parse_rss_image()
└─ Telegram: parse_telegram_image()
     ↓
POLICY VALIDATION
validate_image_policy() → policy checks
     ↓
DRAFT CREATION
├─ primary_image: source image (if valid)
├─ fallback_images: alternatives
└─ ai_image_option: AI generation prompt
     ↓
IMAGE SELECTION
select_image_for_draft() → best option + fallback
     ↓
APPROVAL REQUEST
request_image_approval() → admin review
     ↓
ADMIN DECISION
├─ [✅] Approve source image
├─ [🤖] Generate AI image
└─ [⏭️] Skip image
     ↓
FINAL APPROVAL
(before publication)
```

### Metadata Tracking
```json
{
  "draft_001_image": {
    "url": "original URL",
    "source_type": "telegram",
    "source_name": "@cnc_skill",
    "file_path": "data/image_cache/draft_001_a1b2c3d4.jpg",
    "mime_type": "image/jpeg",
    "size_bytes": 245000,
    "cached_at": "2026-07-14T10:15:00Z",
    "approval_status": "needs_review"
  }
}
```

### Cache Functions
- `cache_image()` - Download & store with metadata
- `get_cache_stats()` - View cache usage (count, size)
- `cleanup_cache()` - Remove images older than N days
- `_load_metadata()` - Load from disk
- `_save_metadata()` - Persist to disk

## 📈 ADMIN WORKFLOW

### Step 1: Draft Created with Images
Admin sees draft with multiple image options:
- Primary: Source image from Telegram/RSS
- Fallbacks: Alternative images
- AI Option: Visual prompt ready for generation

### Step 2: Image Approval Decision
Admin chooses from options:
```
[✅ Use source image] [✅ Use RSS image] [🤖 Generate AI] [⏭️ Skip image]
```

### Step 3: AI Generation (if selected)
- Visual prompt sent to AI service
- ~30 seconds generation time
- Image cached locally
- Returns to step 2 for approval

### Step 4: Final Review
- Content approved/rejected
- Image selected & approved
- Ready for publication

### Example 1: Parse RSS Image
```python
from app.telegram_image_handler import TelegramImageHandler

handler = TelegramImageHandler()

article = {
    "image": "https://example.com/machine.jpg",
    "title": "New CNC Machine",
    "summary": "Latest equipment news"
}

image = handler.parse_rss_image("Woodworking Network", "https://...", article)
success, path = handler.cache_image(image, "draft_001")
```

```python
telegram_media = {
```

```python
source_config = {
```
